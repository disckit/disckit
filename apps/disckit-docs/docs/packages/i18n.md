# @disckit/i18n

Lightweight i18n for Discord bots. Dot-notation key resolution, `{{var}}` interpolation, pluralization, fallback locale, and hot-reload. Zero external dependencies.

```sh
npm install @disckit/i18n
```

[![npm](https://img.shields.io/npm/v/@disckit/i18n?style=flat-square&color=ff468a)](https://www.npmjs.com/package/@disckit/i18n)

## Quick start

```js
const { I18n } = require("@disckit/i18n");

const i18n = new I18n({ fallback: "en" });

i18n.load("en", {
  greet: "Hello, {{name}}!",
  items: { one: "{{n}} item", other: "{{n}} items" },
  errors: { notFound: "Not found." },
});

i18n.load("pt", {
  greet: "Olá, {{name}}!",
  items: { one: "{{n}} item", other: "{{n}} itens" },
});

i18n.t("greet", "en", { name: "Alice" }); // "Hello, Alice!"
i18n.t("greet", "pt", { name: "Alice" }); // "Olá, Alice!"
i18n.t("items", "pt", { n: 3 });          // "3 itens"
i18n.t("errors.notFound", "pt");          // "Not found." (falls back to "en")
```

## I18n

### Constructor

```js
new I18n(options?)
```

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `fallback` | `string` | `"en"` | Locale used when a key is missing in the requested locale. |
| `missing` | `string` | `"__MISSING__{{key}}__"` | Template returned when key is missing in all locales. |

### Methods

#### `load(locale, messages)`

Merges `messages` into the locale (deep merge). Safe to call multiple times with partial message sets.

```js
i18n.load("en", { common: { yes: "Yes" } });
i18n.load("en", { common: { no: "No" } });
// result: { common: { yes: "Yes", no: "No" } }
```

#### `reload(locale, messages)`

Fully **replaces** the locale — does not merge. Use for hot-reload at runtime.

```js
const fresh = JSON.parse(fs.readFileSync("./locales/en.json", "utf8"));
i18n.reload("en", fresh);
```

#### `unload(locale)`

Removes a locale from memory. Returns `true` if it existed.

#### `t(key, locale, vars?)`

Translates a key. Falls back to `fallback` locale if not found. Returns the `missing` template if absent in both.

```js
i18n.t("greet", "es", { name: "Bob" });
// → "Hello, Bob!" (falls back to "en" because "es" not loaded)
```

#### `has(key, locale)`

Returns `true` if the key exists in the requested locale **or** the fallback locale.

#### `locales()`

Returns an array of all loaded locale codes.

### Dot-notation keys

Keys use `.` to traverse nested objects:

```js
i18n.load("en", { errors: { auth: { denied: "Access denied." } } });
i18n.t("errors.auth.denied", "en"); // "Access denied."
```

### Interpolation

Variables use `{{varName}}` syntax. Unknown variables are left unchanged.

```js
i18n.load("en", { welcome: "Welcome to {{server}}, {{user}}!" });
i18n.t("welcome", "en", { server: "My Guild", user: "Alice" });
// "Welcome to My Guild, Alice!"
```

### Pluralization

A key can be an object with `zero`, `one`, and `other` branches. The branch is selected by `vars.n` or `vars.count`.

```js
i18n.load("en", {
  members: { zero: "No members", one: "{{n}} member", other: "{{n}} members" },
});

i18n.t("members", "en", { n: 0 }); // "No members"
i18n.t("members", "en", { n: 1 }); // "1 member"
i18n.t("members", "en", { n: 5 }); // "5 members"
```

## createT

A helper that binds a locale to an `I18n` instance, returning a compact `t(key, vars?)` function.

```js
const { I18n, createT } = require("@disckit/i18n");

const i18n = new I18n({ fallback: "en" });
i18n.load("en", { ping: { reply: "Pong! Latency: {{ms}}ms" } });

// In a command handler
const t = createT(i18n, interaction.locale ?? "en");
await interaction.reply(t("ping.reply", { ms: client.ws.ping }));
```

## Discord bot example

```js
const { I18n, createT } = require("@disckit/i18n");
const { SlashCommandBuilder } = require("discord.js");

const i18n = new I18n({ fallback: "en" });

i18n.load("en", { commands: { info: { title: "Server Info", members: "Members" } } });
i18n.load("pt-BR", { commands: { info: { title: "Informações do Servidor", members: "Membros" } } });

module.exports = {
  data: new SlashCommandBuilder().setName("info").setDescription("Server information"),

  async execute(interaction) {
    const t = createT(i18n, interaction.locale);
    await interaction.reply({
      embeds: [{
        title: t("commands.info.title"),
        fields: [{ name: t("commands.info.members"), value: `${interaction.guild.memberCount}` }],
      }],
    });
  },
};
```
